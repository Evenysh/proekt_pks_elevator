Тестовые сценарии (Практическое занятие №4)
============================================

Список сценариев, раскрывающих основные этапы работы программы:

1. Инициализация здания  
   Описание: создание объекта Building с допустимым числом этажей и лифтов.  
   Реализация: исполняемый файл scenario_building (папка scenarios).

2. Формирование и проверка заявки  
   Описание: создание Request и проверка validate() на корректных и некорректных данных.  
   Реализация: scenario_request.

3. Выбор лифта диспетчером (стратегия «ближайший»)  
   Описание: для фиксированных позиций кабин выбирается лифт с минимальным расстоянием до этажа вызова.  
   Реализация: scenario_strategy.

4. Запуск и остановка симуляции с заявкой  
   Описание: Simulation::start, постановка заявки submitRequest, короткая пауза, Simulation::stop.  
   Реализация: scenario_simulation_lifecycle.

5. Полный интерактивный режим (ручной сценарий)  
   Описание: запуск основного приложения elevator_sim, ввод параметров здания, пункты меню вызова и статуса.  
   Реализация: основной исполняемый файл elevator_sim (не отдельная мини-программа).

Дополнительно: автоматические unit-тесты Catch2 собираются в elevator_tests (каталог tests).

Запуск тестов (из каталога build после cmake):
  ctest --output-on-failure
или:
  ./elevator_tests

Запуск мини-сценариев (из каталога build; бинарники в подпапке scenarios):
  ./scenarios/scenario_building
  ./scenarios/scenario_request
  ./scenarios/scenario_strategy
  ./scenarios/scenario_simulation_lifecycle
